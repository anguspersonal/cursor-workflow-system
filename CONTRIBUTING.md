## What is This Project?

This is a **version-controlled system for managing Cursor RACS** (Rules, Agents, Commands, Skills). It provides:
- A complete workflow from feature ideation to implementation
- Production-quality commands with built-in quality checks
- Battle-tested patterns for testing and development
- A template for organizing Cursor configurations

## How to Contribute

### üêõ Found a Bug in a Command?

1. Open an issue describing:
   - Which command has the issue
   - What you expected to happen
   - What actually happened
   - Steps to reproduce

### üí° Have an Idea for Improvement?

1. Check existing issues to see if it's already suggested
2. Open an issue describing your idea
3. Explain the use case and benefit

### üîß Want to Submit a Command/Rule/Skill?

**We welcome:**
- New commands that fit the workflow
- Improved versions of existing commands
- New templates (rules, skills, agents)
- Better testing patterns
- Documentation improvements

**Guidelines:**
1. Fork the repo
2. Add your contribution to the appropriate directory:
   - Commands ‚Üí `templates/commands/` (for project-specific) or share as an example
   - Rules ‚Üí `templates/rules/`
   - Skills ‚Üí `templates/skills/`
   - Best practices ‚Üí `learnings/best-practices.md`
3. Document it well (what it does, when to use it, examples)
4. Submit a PR with clear description

### üìù Want to Improve Documentation?

Documentation improvements are always welcome! Areas that need help:
- More patterns in `learnings/`
- Better command explanations
- Use case documentation
- Tutorial for getting started

## Code of Conduct

**Be Respectful:**
- This is a learning community
- Everyone's workflow is different
- Constructive feedback only
- Help others grow

**Quality Over Quantity:**
- Well-documented, tested patterns
- Real-world use cases, not theoretical
- Clear explanations with examples

## Questions?

Open an issue with the `question` label. No question is too basic!
